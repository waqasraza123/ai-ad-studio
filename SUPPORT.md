
Support
Where to ask for help

Use the repository in the following way:

bug reports → GitHub Issues

feature requests → GitHub Issues

implementation or usage questions → GitHub Issues with clear context

security concerns → SECURITY.md

What to include

When asking for help, include:

what you are trying to do

what you expected to happen

what actually happened

logs or screenshots when relevant

steps to reproduce when possible

Before opening an issue

Please check whether:

the README already answers the question

an existing issue already covers the same problem

the problem is actually a security issue and should be reported privately

Response expectations

This project may not provide real-time support. Clear, reproducible reports are much easier to triage and resolve.
